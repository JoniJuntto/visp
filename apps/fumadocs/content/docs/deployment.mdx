The versioned templates live in the repository's `deploy/` directory. Keep
machine-specific domains, addresses, and secrets out of Git.

## Relay box

1. Install Tailscale, Caddy, `curl`, and the pinned MediaMTX v1.19.2 binary.
2. Verify the archive against the checksum documented in `deploy/README.md`.
3. Start with `mediamtx.static-auth.yml` and prove a real SRT publish/read using
   temporary credentials.
4. Replace it with `mediamtx.yml` to enable HTTP tenant authentication and hooks.
5. Install the MediaMTX systemd unit and Caddy drop-in.

Expose public UDP 8890, TCP 1935, and TCP 443. Restrict SSH and MediaMTX TCP 9997
to Tailscale, and allow only the app box to reach the Control API. Mirror the
rules in the UpCloud firewall.

## App box

1. Install PostgreSQL, Bun, Tailscale, and Caddy.
2. Fill `/etc/visp/app.env` from `apps/server/.env.example`.
3. Fill `/etc/visp/web.env` from `apps/web/.env.example`; browser variables must
   be present when building the portal.
4. Run `bun install`, `bun run db:migrate`, and `bun run build`.
5. Install the API, web, and Caddy systemd templates.

Register the production Twitch callback as:

```text
https://APP_DOMAIN/api/auth/callback/twitch
```

Expose only public TCP 443. Machine endpoints must receive the relay's direct IP
and must not sit behind a CDN or WAF.

## Rollout order

1. Static MediaMTX publish/read proof.
2. Twitch login and schema migration.
3. MediaMTX HTTP authentication.
4. Path and credential portal.
5. Hooks and reconciliation.
6. RTT guidance and OBS export.

## Maintenance

API and portal restarts do not terminate established streams. A MediaMTX restart
does, so schedule upgrades in a maintenance window. Path provisioning and secret
rotation do not require a relay restart.

## Acceptance test

1. Publish and read one feed over SRT, then repeat over RTMP.
2. Verify dashboard state and ten-second reconciliation.
3. Stop the app while streaming: the stream must survive and a new connection
   must fail.
4. Rotate one publishing device and verify other device URLs still connect;
   rotate the read credential separately and verify only readers are affected.
5. Import the generated scene collection and play every source in OBS.
